* Overview

Storitad is a minimal, privacy-first Android app for recording voice
and video journal entries on a GrapheneOS device (Pixel 9 Pro). It captures
media with structured metadata and stores everything as local files — no
database, no network, no cloud. Entries are later transferred via USB to
sixseven for processing into an org-mode journal.

The app is one half of a two-part system:

1. *Storitad* (this spec) — Android app, capture-only
2. *Storitad Ingest* (separate spec) — sixseven-side script that processes
   captured media into org-mode journal entries with Whisper transcription

** Design Principles

- Capture friction must be minimal — two taps to start recording
- All data stays on-device, no network permissions requested
- Output is plain files on the filesystem, not locked in app-internal storage
- Metadata sidecar format is the contract between Capture and Ingest
- The app should be buildable from source via Nix (reproducible)
- Material You / Material 3 theming, respecting system dark mode
- No Google Play Services dependencies (GrapheneOS-compatible)

* Data Model

** EntryMetadata

#+begin_src kotlin
@Serializable
data class EntryMetadata(
    // Identity
    val id: String,                          // UUID, generated at capture start
    val version: Int = 1,                    // Sidecar format version

    // Temporal
    val capturedAt: Instant,                 // UTC timestamp of recording start
    val duration: Duration,                  // Recording length
    val timezone: String,                    // e.g. "Europe/London"

    // Media
    val mediaFile: String,                   // Relative filename, e.g. "20260414-153200.m4a"
    val mediaType: MediaType,                // VOICE or VIDEO
    val mimeType: String,                    // e.g. "audio/mp4", "video/mp4"

    // Journal metadata (user-provided)
    val subject: String,                     // Brief title / what this is about
    val recipients: List<String> = listOf("family"),
                                             // Who this is for: "casper", "family", "general"
    val mood: String? = null,                // Optional: "happy", "reflective", "proud", etc.
    val tags: List<String> = emptyList(),    // Free-form tags
    val notes: String? = null,               // Additional context, free text

    // Device context (auto-captured)
    val device: String = Build.MODEL,        // e.g. "Pixel 9 Pro"
    val appVersion: String,                  // Storitad version string

    // Transcription (optional, phase 2)
    val transcriptDraft: String? = null,     // On-device Whisper output
    val transcriptModel: String? = null,     // e.g. "whisper-tiny.en"

    // Processing status (used by Ingest, written by Capture)
    val processed: Boolean = false           // Ingest sets true after import
)

enum class MediaType { VOICE, VIDEO }
#+end_src

* Recording Specifications

** Voice Recording

| Parameter    | Value                     |
|--------------+---------------------------|
| Format       | AAC in M4A container      |
| Sample rate  | 44100 Hz                  |
| Channels     | Mono                      |
| Bitrate      | 128 kbps                  |
| Max duration | 30 minutes (configurable) |

AAC/M4A chosen over WAV for storage efficiency while maintaining good
quality. Whisper handles M4A natively via ffmpeg on the ingest side.

** Video Recording

| Parameter    | Value                      |
|--------------+----------------------------|
| Format       | H.264 in MP4 container     |
| Resolution   | 1080p (front camera)       |
| Frame rate   | 30 fps                     |
| Audio        | AAC stereo, 128 kbps       |
| Max duration | 10 minutes (configurable)  |
| Camera       | Front-facing by default    |

Front camera default because these are personal messages to your kids —
they want to see your face. Flip-to-rear available for showing something.

* Permissions

Only what is strictly necessary. No network permission at all.

| Permission                    | Reason                            |
|-------------------------------+-----------------------------------|
| =RECORD_AUDIO=                | Voice and video recording         |
| =CAMERA=                      | Video recording                   |
| =WRITE_EXTERNAL_STORAGE=      | Saving to Documents (pre-API 30) |
| =MANAGE_EXTERNAL_STORAGE=     | Or use SAF for directory access   |
| =FOREGROUND_SERVICE=          | Recording continues if app backgrounded |
| =POST_NOTIFICATIONS=          | Recording-in-progress notification |

Explicitly NOT requested:
- =INTERNET= / =ACCESS_NETWORK_STATE= — no network access at all
- =ACCESS_FINE_LOCATION= / =ACCESS_COARSE_LOCATION= — no location tracking
- Any Google Play Services permissions

* USB Transfer Workflow

When the Pixel is connected to sixseven via USB:

#+begin_src bash
# On sixseven — pull all pending captures
# (assuming MTP mount or adb)

# Option A: adb pull
adb pull /storage/emulated/0/Documents/Storitad/inbox/ \
         ~/journal/inbox/

# Option B: MTP via gio (GNOME) or jmtpfs
jmtpfs /mnt/pixel
cp -r /mnt/pixel/Documents/Storitad/inbox/* ~/journal/inbox/
fusermount -u /mnt/pixel

# Option C: Script it (recommended)
# ~/bin/storitad-pull — a small wrapper that handles mount + copy + verify
#+end_src

After successful transfer, the ingest script on sixseven processes the
inbox and the user can optionally archive/delete from the phone.

* Ingest Script (sixseven side) — Summary

Separate deliverable, but the interface contract is:

1. Reads =*.json= sidecar files from =~/journal/inbox/=
2. For each entry:
   a. Runs =whisper.cpp= on the media file (or uses =transcriptDraft= if present)
   b. Generates an org-mode entry
   c. Appends to =~/journal/journal.org= (or datetree structure)
   d. Moves media to =~/journal/media/YYYY/MM/=
   e. Moves sidecar to =~/journal/processed/=
3. Org entry format:

#+begin_src org
,* Picking Casper up from school :voicenote:casper:
:PROPERTIES:
:ID:        a1b2c3d4-e5f6-7890-abcd-ef1234567890
:CAPTURED:  [2026-04-14 Tue 15:32]
:DURATION:  2m 34s
:MEDIA:     [[file:media/2026/04/20260414-153200-voice.m4a]]
:MOOD:      happy
:DEVICE:    Pixel 9 Pro
:END:

Just picked Casper up from school. He was buzzing about
beating some impossible level in Geometry Dash. He's been
stuck on it for weeks and finally nailed it today...

** Goal

Deliver the Phase 1 MVP of Storitad — a GrapheneOS-compatible Android app that
records voice entries, attaches structured metadata, writes a JSON sidecar
alongside the media file in =Documents/Storitad/inbox/=, and lets the user
browse/play prior captures.

** Architecture

Single-module Kotlin Android app. Jetpack Compose + Material 3 UI,
=navigation-compose= for screen flow, =MediaRecorder= for audio capture,
=MediaPlayer= for playback, app-scoped external storage under
=Documents/Storitad/inbox/= (Phase 4 migrates to public MediaStore
collection). Metadata serialised via =kotlinx.serialization= as
=<basename>.json= sidecars. No DI framework, no Room, no network deps.
State held in plain Composable state + a small in-memory =DraftHolder=
singleton for the capture → review → metadata handoff.

** Tech Stack

Kotlin 2.0.21, Gradle 8.10.2 (Kotlin DSL), AGP 8.7.3, Compose BOM 2024.12.01,
Material 3, =navigation-compose= 2.8.5, =kotlinx-serialization-json= 1.7.3,
=kotlinx-datetime= 0.6.1, =accompanist-permissions= 0.36.0. CameraX deps
pre-declared in the version catalog but unused in Phase 1.

fun StoritadTheme(
    darkTheme: Boolean = isSystemInDarkTheme(),
    dynamicColor: Boolean = true,
    content: @Composable () -> Unit
) {

        else -> LightColors
    }
    MaterialTheme(colorScheme = scheme, typography = Typography, content = content)
}

package uk.storitad.capture.metadata

import kotlinx.serialization.json.Json
import java.io.File

class MetadataRepository(private val inboxDir: File) {

    private val json = Json {
        prettyPrint = true
        encodeDefaults = true
        ignoreUnknownKeys = true
    }

    init { inboxDir.mkdirs() }

    fun write(entry: EntryMetadata) {
        val basename = entry.mediaFile.substringBeforeLast('.')
        val file = File(inboxDir, "$basename.json")
        file.writeText(json.encodeToString(EntryMetadata.serializer(), entry))
    }

    fun read(basename: String): EntryMetadata {
        val file = File(inboxDir, "$basename.json")
        return json.decodeFromString(EntryMetadata.serializer(), file.readText())
    }

    fun list(): List<EntryMetadata> =
        inboxDir.listFiles { f -> f.isFile && f.name.endsWith(".json") }
            ?.mapNotNull {
                runCatching {
                    json.decodeFromString(EntryMetadata.serializer(), it.readText())
                }.getOrNull()
            }
            ?.sortedByDescending { it.capturedAt }
            ?: emptyList()

    fun mediaFile(entry: EntryMetadata): File = File(inboxDir, entry.mediaFile)

    fun delete(entry: EntryMetadata) {
        val basename = entry.mediaFile.substringBeforeLast('.')
        File(inboxDir, "$basename.json").delete()
        File(inboxDir, entry.mediaFile).delete()
    }
}

package uk.storitad.capture.capture

import android.content.Context
import android.media.MediaRecorder
import android.os.Build
import java.io.File

class AudioRecorder(private val context: Context) {
    private var recorder: MediaRecorder? = null
    private var outputFile: File? = null
    private var startedAtMs: Long = 0

    fun start(targetFile: File) {
        require(recorder == null) { "recorder already running" }
        outputFile = targetFile
        val r = if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.S) {
            MediaRecorder(context)
        } else {
            @Suppress("DEPRECATION") MediaRecorder()
        }
        r.setAudioSource(MediaRecorder.AudioSource.MIC)
        r.setOutputFormat(MediaRecorder.OutputFormat.MPEG_4)
        r.setAudioEncoder(MediaRecorder.AudioEncoder.AAC)
        r.setAudioChannels(1)
        r.setAudioSamplingRate(44100)
        r.setAudioEncodingBitRate(128_000)
        r.setOutputFile(targetFile.absolutePath)
        r.prepare()
        r.start()
        startedAtMs = System.currentTimeMillis()
        recorder = r
    }

    fun stop(): Long {
        val r = recorder ?: error("recorder not running")
        return try {
            r.stop()
            System.currentTimeMillis() - startedAtMs
        } finally {
            r.release()
            recorder = null
        }
    }

    fun cancel() {
        val r = recorder ?: return
        runCatching { r.stop() }
        r.release()
        recorder = null
        outputFile?.delete()
        outputFile = null
    }
}

package uk.storitad.capture.ui.drafts

import kotlinx.datetime.Instant
import kotlinx.datetime.TimeZone
import java.io.File

object DraftHolder {
    data class Draft(
        val basename: String,
        val capturedAt: Instant,
        val timezone: TimeZone,
        val mediaFile: File,
        var durationMs: Long = 0
    )

    @Volatile private var current: Draft? = null

    fun begin(basename: String, at: Instant, zone: TimeZone, mediaFile: File) {
        current = Draft(basename, at, zone, mediaFile)
    }

    fun finalise(durationMs: Long) { current?.durationMs = durationMs }

    fun get(): Draft? = current

    fun clear() {
        current?.mediaFile?.takeIf { it.exists() }?.delete()
        current = null
    }

    fun consume() { current = null }
}

import android.app.Notification

import android.app.NotificationManager
import android.app.Service

import android.os.Binder
import android.os.Build
import android.os.IBinder

import uk.storitad.capture.R

class RecordingService : Service() {

    inner class LocalBinder : Binder() { val service get() = this@RecordingService }

    private val binder = LocalBinder()
    override fun onBind(intent: Intent?): IBinder = binder

    override fun onStartCommand(intent: Intent?, flags: Int, startId: Int): Int {
        ensureChannel()
        startForeground(NOTIF_ID, buildNotification("Recording…"))
        return START_NOT_STICKY
    }

    fun updateElapsed(ms: Long) {
        val mgr = getSystemService(Context.NOTIFICATION_SERVICE) as NotificationManager
        mgr.notify(NOTIF_ID, buildNotification(formatElapsed(ms)))
    }

    private fun ensureChannel() {
        val mgr = getSystemService(Context.NOTIFICATION_SERVICE) as NotificationManager
        if (mgr.getNotificationChannel(CHANNEL) == null) {
            mgr.createNotificationChannel(
                NotificationChannel(CHANNEL, "Recording", NotificationManager.IMPORTANCE_LOW)
            )
        }
    }

    private fun buildNotification(text: String): Notification =
        NotificationCompat.Builder(this, CHANNEL)
            .setSmallIcon(android.R.drawable.ic_btn_speak_now)
            .setContentTitle("Storitad")
            .setContentText(text)
            .setOngoing(true)
            .build()

    private fun formatElapsed(ms: Long): String {
        val sec = ms / 1000; return "%d:%02d".format(sec / 60, sec % 60)
    }

    companion object {
        const val CHANNEL = "storitad.recording"
        const val NOTIF_ID = 1001
    }
}

package uk.storitad.capture.capture

import android.content.Context
import androidx.camera.core.CameraSelector
import androidx.camera.lifecycle.ProcessCameraProvider
import androidx.camera.video.FileOutputOptions
import androidx.camera.video.Quality
import androidx.camera.video.QualitySelector
import androidx.camera.video.Recorder
import androidx.camera.video.Recording
import androidx.camera.video.VideoCapture
import androidx.lifecycle.LifecycleOwner
import java.io.File
import java.util.concurrent.ExecutorService
import java.util.concurrent.Executors

class VideoRecorder(private val context: Context) {
    private val executor: ExecutorService = Executors.newSingleThreadExecutor()
    private lateinit var videoCapture: VideoCapture<Recorder>
    private var active: Recording? = null
    private var startedAtMs = 0L
    var useFrontCamera = true

    suspend fun bind(owner: LifecycleOwner, preview: androidx.camera.core.Preview) {
        val provider = ProcessCameraProvider.getInstance(context).get()
        val recorder = Recorder.Builder()
            .setQualitySelector(QualitySelector.from(Quality.HD))
            .build()
        videoCapture = VideoCapture.withOutput(recorder)
        val selector = if (useFrontCamera) CameraSelector.DEFAULT_FRONT_CAMERA
                       else CameraSelector.DEFAULT_BACK_CAMERA
        provider.unbindAll()
        provider.bindToLifecycle(owner, selector, preview, videoCapture)
    }

    @androidx.annotation.RequiresPermission(allOf = [
        android.Manifest.permission.RECORD_AUDIO, android.Manifest.permission.CAMERA
    ])
    fun start(target: File) {
        val opts = FileOutputOptions.Builder(target).build()
        active = videoCapture.output
            .prepareRecording(context, opts)
            .withAudioEnabled()
            .start(executor) { /* event callbacks: finalize, error */ }
        startedAtMs = System.currentTimeMillis()
    }

    fun stop(): Long {
        active?.stop()
        active = null
        return System.currentTimeMillis() - startedAtMs
    }

    fun cancel() { active?.stop(); active = null }
}

#+begin_src kotlin
class RecipientsRepository(private val ctx: Context) {
    private val file get() = File(ctx.filesDir, "recipients.json")
    private val json = Json { prettyPrint = true; encodeDefaults = true }

    fun list(): List<Recipient> = ensureSeeded().let {
        json.decodeFromString(RecipientsConfig.serializer(), file.readText()).recipients
    }

    fun save(items: List<Recipient>) {
        file.writeText(json.encodeToString(
            RecipientsConfig.serializer(), RecipientsConfig(items)
        ))
    }

    private fun ensureSeeded() {
        if (file.exists()) return
        val text = ctx.assets.open("recipients.json").bufferedReader().use { it.readText() }
        file.writeText(text)
    }
}

** Recipient aliases

The =recipients= field in the sidecar is semantically a *tag set* — a flat
list of ids that combines real people and scope-aliases. No schema change,
no sidecar version bump.

*** Canonical ids

| id      | kind   | expansion on ingest           |
|---------+--------+-------------------------------|
| griffin | person | (none — leaf)                 |
| julian  | person | (none — leaf)                 |
| casper  | person | (none — leaf)                 |
| emma    | person | (none — leaf)                 |
| family  | alias  | =[griffin, julian, casper, emma]= |
| general | scope  | (none — means "not directed") |

The capture app never expands aliases. It writes the ids the user selected
verbatim, so round-tripping is lossless:

- Picking just the =family= chip writes =recipients: ["family"]=.
- Picking =casper= and =emma= writes =recipients: ["casper", "emma"]=.
- Picking =general= writes =recipients: ["general"]= (explicitly untargeted).

*** Ingest-side contract

Storitad Ingest (sixseven-side, not in this repo) is responsible for:

1. Expanding aliases at journal-write time: if =family= is present, emit org
   tags for each expanded person id *in addition to* the =family= tag so the
   entry is reachable from both angles.
2. Treating =general= as a hint to omit per-person tags (the entry still
   gets its type tag like =:voicenote:=).
3. Any future alias (e.g. =boys= → =[griffin, julian, casper]=) is declared
   in the ingest configuration, not on the phone. The phone is allowed to
   write unknown ids; the ingest script falls back to tagging verbatim.

*** App behaviour

- Settings lets the user add any id (person or alias) with lowercase-ASCII
  validation. The app does not know the difference.
- Metadata chips are shown in the order recipients.json declares; adding a
  new alias means appending it to that file.
- No UI distinction between "person" and "alias" in Phase 2. If that becomes
  useful later, add a =kind= field on =Recipient= (default =person=) and
  style accordingly — fully backwards compatible.
